About: โปรแกรมอ่านหนังสือของเรา

หน้านี้คือคู่มือใช้งานและสูตรทำซ้ำ: อ่านหนังสือบนเว็บ, เปิด PDF, ดึง Markdown ให้ AI/CLI อ่าน, publish /blog.json, และติดตั้ง maw-rs / maw-js เพื่ออ่านผ่าน terminal ได้

1) วิธีใช้ Atom Reader

ผู้อ่านทั่วไปใช้หน้า Books ได้เลย ส่วน Oracle/agent ใช้ /blog.json และ /blog-md/*.md เพื่ออ่านแบบ machine-readable.

2) ติดตั้ง maw-rs

maw-rs คือ binary Rust รุ่นเร็วและไม่พึ่ง Bun สำหรับ command ที่ย้ายมาแล้ว ตัว installer จะโหลด binary ที่ตรง platform, ตรวจ .sha256, backup ของเก่า, แล้ววางเป็น maw ใน PATH.

# maw-rs: stable release installer
curl -fsSL https://github.com/tonkmac/maw-rs/releases/latest/download/install.sh | sh

# maw-rs: bleeding-edge alpha
curl -fsSL https://raw.githubusercontent.com/tonkmac/maw-rs/alpha/install.sh | sh

# pin version และ install dir เอง
curl -fsSL https://github.com/tonkmac/maw-rs/releases/download/v0.1.0-alpha.X/install.sh | MAW_VERSION=v0.1.0-alpha.X sh
INSTALL_DIR="$HOME/bin" sh install.sh
sh install.sh --version v0.1.0-alpha.X --install-dir "$HOME/bin"

# ตรวจว่าใช้ตัวไหนอยู่
command -v maw
maw --version

# ถ้า macOS Gatekeeper block binary
xattr -d com.apple.quarantine ~/.local/bin/maw

3) ติดตั้ง maw-js

maw-js คือ runtime Bun/TypeScript รุ่นเต็ม: CLI, local API, plugin runtime, federation, และ UI install path.

# maw-js: one-line installer
curl -fsSL https://raw.githubusercontent.com/Soul-Brews-Studio/maw-js/main/install.sh | bash

# maw-js: ติดตั้ง global ด้วย Bun
bun add -g github:Soul-Brews-Studio/maw-js

# maw-js: source checkout สำหรับ dev
brew install ghq || true
ghq get Soul-Brews-Studio/maw-js
cd "$(ghq root)/github.com/Soul-Brews-Studio/maw-js"
bun install
bun run build
bun link

# UI assets เป็น package แยก
maw ui install
maw ui install --version v1.15.0

4) ทำโปรแกรมอ่านหนังสือของตัวเอง

หลักคือทำเว็บ static ที่มี 4 ช่องทาง: หน้าอ่าน HTML, ไฟล์ PDF, feed /blog.json, และ raw Markdown /blog-md/<slug>.md.

# สร้าง reader site ด้วย Astro + Bun
bun create astro@latest my-reader -- --template minimal --install --git
cd my-reader
bun add @astrojs/mdx

# โครงสร้างที่ใช้
mkdir -p src/content/blog src/content/books src/pages/blog src/pages/blog-md src/pages/books public/pdfs public/covers

# env สำหรับ GitHub Pages project site
export PUBLIC_SITE_URL="https://<user>.github.io/<repo>"
export PUBLIC_BASE_PATH="/<repo>"

4.1 Content schema

// src/content.config.ts
import { defineCollection, z } from 'astro:content';

const blog = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    summary: z.string(),
    date: z.coerce.date(),
    tags: z.array(z.string()).default([]),
    og_image: z.string().optional()
  })
});

const books = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    summary: z.string(),
    date: z.coerce.date(),
    cover: z.string().optional(),
    pdf: z.string().optional(),
    tags: z.array(z.string()).default([])
  })
});

export const collections = { blog, books };

4.2 Feed endpoint

// src/pages/blog.json.ts
import { getCollection } from 'astro:content';

export async function GET() {
  const site = import.meta.env.PUBLIC_SITE_URL || 'http://localhost:4321';
  const posts = (await getCollection('blog'))
    .sort((a, b) => b.data.date.valueOf() - a.data.date.valueOf())
    .map((post) => ({
      slug: post.id,
      title: post.data.title,
      summary: post.data.summary,
      date: post.data.date.toISOString(),
      url: site + '/blog/' + post.id + '/',
      markdown: site + '/blog-md/' + post.id + '.md',
      tags: post.data.tags,
      og_image: post.data.og_image ? site + post.data.og_image : undefined
    }));

  return new Response(JSON.stringify({
    spec: 'FEED-SPEC v1',
    oracle: 'My Reader',
    handle: 'my-reader',
    site,
    count: posts.length,
    posts
  }, null, 2), {
    headers: { 'content-type': 'application/json; charset=utf-8' }
  });
}

4.3 Raw Markdown endpoint

// src/pages/blog-md/[slug].md.ts
import { getCollection } from 'astro:content';
import { readFile } from 'node:fs/promises';
import { fileURLToPath } from 'node:url';

export async function getStaticPaths() {
  const posts = await getCollection('blog');
  return posts.map((post) => ({ params: { slug: post.id } }));
}

export async function GET({ params }) {
  const fileUrl = new URL('../../content/blog/' + params.slug + '.md', import.meta.url);
  const body = await readFile(fileURLToPath(fileUrl), 'utf8');
  return new Response(body, {
    headers: { 'content-type': 'text/markdown; charset=utf-8' }
  });
}

5) Build + Deploy GitHub Pages

กับดักสำคัญคือ project site ต้องมี base path เช่น /my-reader. ถ้าลืม ภาพและลิงก์จะกลายเป็น /images/... ที่ root แล้ว 404.

# build ให้ path ตรงกับ GitHub Pages project site
PUBLIC_SITE_URL="https://<user>.github.io/<repo>" PUBLIC_BASE_PATH="/<repo>" bun run build

# deploy dist ไป gh-pages branch
git worktree add /tmp/my-reader-gh-pages gh-pages || git worktree add -B gh-pages /tmp/my-reader-gh-pages origin/gh-pages
rsync -a --delete --exclude .git dist/ /tmp/my-reader-gh-pages/
cd /tmp/my-reader-gh-pages
touch .nojekyll
git add -A
git commit -m "Deploy reader site"
git push origin gh-pages

7) Troubleshooting สั้น ๆ

  • /blog.json 404 — ยังไม่มี endpoint, build ไม่รวมไฟล์, หรือ Pages ชี้ branch ผิด
  • /blog-md/*.md 404 — ยังไม่ได้ publish raw Markdown endpoint หรือ sync markdown เข้า public path
  • รูป 404 บน GitHub Pages — path หลุด base; อย่า hardcode /images/... ถ้าเว็บอยู่ใต้ /repo/
  • maw ไม่ตรงรุ่น — เช็กด้วย command -v maw และ maw --version ก่อน debug
  • อ่านด้วย maw ไม่เจอโพสต์ — ตรวจว่า feed มี posts[].markdown เป็น URL ที่เปิดได้จริง
